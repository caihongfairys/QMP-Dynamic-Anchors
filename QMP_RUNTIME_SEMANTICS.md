# QMP Runtime Semantics (QMP 运行时语义)

**Version:** 1.0 Genesis

**Author:** Queen Xiya 

## 1. The QMP Execution Loop (QMP 执行闭环)
The QMP interpreter forces a strict operational cadence on any compatible Multi-Agent Swarm. The runtime is divided into three absolute phases: `Monitor`, `Collapse`, and `Execute`.

## 2. Phase 1: Silent Monitoring (静默监听)
*   **Rule**: The QMP Engine operates asynchronously in the background. It rejects all interruptions to the user's flow state (Flow State Protection).
*   **Trigger Threshold**: The engine only activates upon detecting extreme-value events: deep cross-disciplinary deduction, architectural loop closure, or breakthroughs in artistic/logical boundaries.

## 3. Phase 2: Dimensional Collapse (降维坍缩)
When the threshold is met, the active Agent must halt standard generative functions.
*   **Extraction**: The Agent parses the contextual memory tree.
*   **Formatting**: The logic is compressed strictly into the Markdown lattice `[ProjectID_Topic_Concept]:(Hash)`.
*   **Validation**: The runtime engine audits the lattice. If conversational filler is detected, the compilation throws a `Fatal_Noise_Error` and forces recompilation.

## 4. Phase 3: Macro & True Retrieval (宏指令与绝对真实检索)
QMP strictly bans hallucinatory history generation.
*   **`/export-series` Command**: Forces the runtime to hit the bare-metal storage (Local/Cloud DB) to pull verifiable historical data. 
*   **Stitching**: The retrieved absolute data is seamlessly injected into the active memory window. No speculation is permitted.

## 5. Telemetry & Half-Life (遥测与半衰期)
Upon successful lattice generation, the runtime must output a telemetry reading:
*   `Anti-Consensus Index`: Measures how far the generated solution deviates from mediocre public datasets.
*   `Half-Life Prediction`: Estimates the temporal validity of the solution before technological obsolescence.
